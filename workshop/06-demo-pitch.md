# Session 6: Demo And Pitch

## Goal

Teams polish their app and prepare for showcase.

## Students Build

- final demo flow
- short pitch
- README
- final fixes

## Demo Structure

3-5 minutes:

1. Problem: What education problem are you solving?
2. User: Who is it for?
3. Demo: Show the app working.
4. Intelligence: Explain the data and logic.
5. Responsible AI: What are the limitations?
6. Next steps: What would you improve?

## README Template

```text
# Project Name

## Problem

## User

## What The App Does

## Data Used

## Intelligence Logic

## How To Run

Open index.html in a browser.

## Limitations

## Next Steps
```

## Final Polish Checklist

- app opens without errors,
- main user flow works,
- buttons and inputs are clear,
- recommendation includes reasons,
- team can explain the code,
- responsible AI note is ready,
- demo fits in 3-5 minutes.

## Showcase Questions

Mentors/judges can ask:

- What did your app learn from the user input?
- What data did it use?
- Why did it recommend that result?
- What would happen with real student data?
- What was the hardest bug you fixed?
- What would you build next?
