# Demo Rubric

Use this with the [Mentor Guide Index](./README.md), [Student
Handout](../student/handout.md), and [Session 6: Demo And
Pitch](../workshop/sessions/06-demo-pitch.md).

Score each category from 1 to 4.

| Category | 1 | 2 | 3 | 4 |
|---|---|---|---|---|
| Problem and user | Unclear | Some user/problem | Clear user/problem | Strong local relevance |
| Working web app | Mostly broken | Some screens work | Core flow works | Smooth, polished MVP |
| Data | No clear data | Small or unclear data | Useful dummy dataset | Well-structured data with topic tags/categories |
| Intelligence layer | No intelligence | Basic hardcoded output | Clear recommendation/scoring/retrieval logic | Strong logic with explanation and edge cases |
| Integration | Separate from app | Partly connected | User input changes output | Feels natural in the user flow |
| Responsible AI | Not mentioned | Basic limitation | Clear limitations and risks | Thoughtful bias/testing/accuracy discussion |
| AI assistant/code understanding | Cannot explain code | Explains parts | Explains main functions | Explains choices, bugs, and improvements |
| Demo and pitch | Hard to follow | Basic demo | Clear 3-5 minute demo | Compelling story and next steps |

## Minimum MVP Bar

A team is demo-ready if:

- app opens in browser,
- user can complete one core flow,
- app produces an intelligent recommendation/result,
- team can explain the data and logic,
- team can name at least two limitations.

## Suggested Awards

- Most useful education solution
- Best responsible AI explanation
- Best user experience
- Best use of data
- Best technical improvement
- Best pitch
